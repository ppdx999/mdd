# システム設計ドキュメント

## 概要

本ドキュメントでは認証システムの設計を記述する。

## ユースケース図

```usecase
actor ユーザー
actor 管理者

package "認証" {
  usecase ログイン
  usecase ログアウト
  usecase パスワードリセット
}

ユーザー -> ログイン
ユーザー -> ログアウト
ユーザー -> パスワードリセット
管理者 -> ログイン
```

## データフロー図

```dfd
entity ユーザー
entity メールサーバー

process 認証処理
process パスワードリセット

datastore ユーザーマスタ {
  ユーザーID
  メールアドレス
  パスワードハッシュ
  最終ログイン日時
}

datastore セッション {
  セッションID
  ユーザーID
  有効期限
}

ユーザー -> 認証処理 : "認証情報"
認証処理 -> ユーザーマスタ : "認証照会"
ユーザーマスタ -> 認証処理 : "認証結果"
認証処理 -> セッション : "セッション発行"
ユーザー -> パスワードリセット : "リセット要求"
パスワードリセット -> ユーザーマスタ : "パスワード更新"
パスワードリセット -> メールサーバー : "リセットメール"
```

## 補足

- 認証にはJWTトークンを使用
- パスワードはbcryptでハッシュ化
- セッションの有効期限は24時間
